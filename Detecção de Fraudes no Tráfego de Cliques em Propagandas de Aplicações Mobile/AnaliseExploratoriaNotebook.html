Descries sobre as colunas:

ip: endereo ip do click
app: id do aplictivo para marketing
device: tipo do celular, ex: Iphone 6 plus, Iphone 7 plus, Hawuei mate 7, etc
os: verso do so do aparelho celular
channel: id do canal do editor de anncios para celular
click_time: hora do click no anncio(utc)
attributed_time: se o usurio fez o download do app depois de clicar no ad, hora do download do app
is_attributed: varivel alvo a ser predita, indicando se o app foi baixado
OBS: ip, app, device, os e channel so dados encriptografados.

#.

Analisando o grfico, a linha de tendncia(representada pela linha azul), est decrescente, isso quer dizer que, no decorrer do dia, o volume de cliques no anncio tende a diminuir.
Observa-se um pico de volume de acesso s 4 horas.
O volume de cliques varia, no entanto se mantm em alta at s 14 horas, aps, o volume de acessos vai caindo at o vale(que o perodo com menor volume de acessos), s 20 horas.
<br

Concluso: O volume de cliques e download variam durante as horas, porm quando chega s 14 horas, o volume cai drasticamente, provavelmente, o volume de pessoas online est menor.
O momento ideal para se fazer alguma campanha seria aps s 0 horas, at s 14, pois o volume de pessoas e pessoas convertidas(download do app), so maiores.

Qual a distribuio varivel app quando is_attributed == 1?

# Construindo histograma
app <- df %>% filter(is_attributed == 1) %>% select(app)
g1 <- ggplot(app, aes(x = app)) +
  geom_histogram(aes(y = ..density..), binwidth = 2, color = "#BFD7F2", position = "identity") +
  theme_bw() + 
  xlab("app") +
  geom_density(alpha = .5, fill = "#375270") +
   geom_vline(aes(xintercept=mean(app)),
             color="#5C89BD", linetype="dashed", size=1, alpha = .3) +
  ggtitle("Distribuio da varvel app quando is_attributed == 1") +
  theme(plot.title = element_text(hjust = 0.5, size = 17, face = "bold"))

g1

order_app <- df %>% filter(is_attributed == 1) %>% group_by(app) %>% summarise(count = n())

order_app <- order_app[order(order_app$percent, decreasing = TRUE),]
